What is a Pushdown Automaton?

A pushdown automaton (PDA) is a formal computation model—imagine a standard finite state machine, but augmented with an infinite stack memory. This leap in architecture allows it to recognize **Context-Free Languages**, granting it the ability to count, mirror, and match nested structures, something no DFA or NFA can ever achieve.

Context-Free Boundaries

Why do we need a stack? Consider the language aⁿbⁿ. To ensure an equal number of A's and B's, a finite automaton would need infinite states. A PDA simply pushes a marker for every 'a', and pops a marker for every 'b'. If the stack runs empty precisely as the input ceases, the string is accepted. It learns to balance.

Limits of the Machine

While a single stack grants immense power, it is strictly bound. A PDA cannot parse aⁿbⁿcⁿ. A single Last-In-First-Out (LIFO) stack cannot simultaneously track two independent counts. To compute that, you must ascend to a Turing Machine. Here, you visualize exactly where that boundary lies.
