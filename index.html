What makes a PDA different from a finite automaton?

A PDA adds an unbounded last-in-first-out stack. This extra memory lets it count and match nested structures that no DFA or NFA can handle — a fundamental leap in computational power.

PDA Theory — Quick Guide

What is a Pushdown Automaton?

A pushdown automaton (PDA) is a finite automaton augmented with a stack — an unbounded last-in-first-out memory structure. This extra resource enables recognition of context-free languages, a strictly larger class than regular languages.

📼

Input Tape

Symbols read left → right, one at a time. ε-transitions read nothing.

🔘

State Control

Finite set of states. One active at a time. Some states accept, some reject.

📚

Stack Memory

Push symbols on top, pop from top. This is the PDA's key superpower over a DFA.

Why aⁿbⁿ Needs a Stack

No DFA can count arbitrarily. For aⁿbⁿ, push one marker per a, pop one per b. Accept if the stack empties exactly as input ends. The stack stores the count.

Why wwᴿ is Context-Free

Push the first half of the string, then pop while matching the mirrored second half. A stack naturally stores a sequence in reverse order for later comparison.

Why aⁿbⁿcⁿ is NOT Context-Free

A single stack cannot simultaneously track two independent counts. By the pumping lemma for CFLs, this language provably requires more computational power than any PDA provides — you need a Turing machine.

Transition Notation

Each transition: (state, read, pop) → (newState, push). In a DPDA (deterministic), at most one transition fires per configuration. This simulator takes the first matching transition.

Acceptance

This simulator uses accept-by-final-state: the machine accepts if it reaches a designated accepting state with no remaining input. The stack contents at that point are irrelevant.
